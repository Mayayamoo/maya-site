CSS Integration Conflicts Analysis
=====================================

## Overview
Consolidated `quirky-bubble-mobile.css`, `timeline-toc.css`, and optimized `work.js` for better maintainability and reduced file overlap.

## Files Analyzed
1. `css/quirky-bubble-mobile.css` - Mobile timeline bubble card styles
2. `css/timeline-toc.css` - Table of contents styles with duplicated blocks
3. `js/work.js` - Timeline JavaScript functionality
4. `css/work.css` - Main timeline styles
5. `endeavours.html` - HTML file importing these resources

## Conflicts Found & Resolutions

### 1. Mobile Table of Contents Visibility
**Conflict**: Table of contents was showing on mobile instead of being hidden
**Location**: `css/timeline-toc.css:136-169`
**Resolution**: Simplified mobile media query to use `display: none !important` instead of repositioning styles

### 2. CSS Duplication in timeline-toc.css
**Conflict**: Lines 1-139 and 140-298 contained identical CSS blocks with slight variations
**Location**: `css/timeline-toc.css:1-298`
**Resolution**: Removed duplicate CSS blocks (lines 140-298), kept the first complete block with enhanced focus/active states

### 2. Mobile Timeline Style Overlap
**Conflict**: `quirky-bubble-mobile.css` and `work.js` created mobile timeline elements that overlapped with existing desktop styles
**Location**: 
- `css/quirky-bubble-mobile.css:1-585` (entire file)
- `css/work.css:853-1030` (existing mobile styles)
**Resolution**: 
- Consolidated all mobile timeline styles from `quirky-bubble-mobile.css` into `work.css` within the existing `@media (max-width: 768px)` block
- Created comprehensive mobile timeline styles including:
  - `.mobile-timeline-active` class system
  - Quirky bubble card designs with corner decorations
  - Mobile filter bar and buttons
  - Timeline track and connector dots
  - Category-specific color schemes

### 3. Redundant CSS Import
**Conflict**: `endeavours.html` imported `css/quirky-bubble-mobile.css` which is no longer needed
**Location**: `endeavours.html:33`
**Resolution**: Removed the import line `<link rel="stylesheet" href="css/quirky-bubble-mobile.css">` from HTML

### 4. JavaScript Mobile Timeline Logic
**Conflict**: `work.js` contained both desktop and mobile timeline creation logic that could conflict
**Location**: `js/work.js:518-642` (createMobileTimelineBubbles function)
**Resolution**: 
- Maintained existing mobile timeline creation logic in `work.js`
- Ensured CSS classes match between JS creation and consolidated CSS styles
- No changes needed to JavaScript as it properly creates elements with correct classes

## Benefits of Integration

### File Reduction
- Eliminated `css/quirky-bubble-mobile.css` (585 lines)
- Cleaned up `css/timeline-toc.css` (removed 158 duplicate lines)
- Reduced HTTP requests from 4 to 3 CSS files

### Style Consistency
- Unified mobile timeline styles under single media query
- Consistent color variables and design patterns
- Eliminated potential cascade conflicts

### Maintainability

- Easier to update responsive breakpoints
- Reduced code duplication
